# План: Push-уведомления при заблокированном экране (Web Push)

## Статус: Отложено (не первоочередное, обсуждено)

## Цель

Настоящие пуш-уведомления на телефон, которые приходят, **даже когда экран
заблокирован, а приложение свёрнуто/закрыто**. Для мобильной админки
(менеджеры/диспетчеры) и мобильной страницы водителя.

## Текущее состояние (что уже есть)

- В `app/mobile/driver/components/DriverDashboard.tsx` используется браузерный
  `Notification` API (`showPushNotification`, `requestNotifPermission`).
- **Ограничение:** работает только пока страница открыта и активна. При
  заблокированном экране / свёрнутом приложении не срабатывает.
- Нет `manifest.json`, нет service worker, нет подписки Web Push, нет отправки
  пушей с сервера.

## Что нужно реализовать (4 части)

### 1. PWA-установка
- `public/manifest.json` (имя, иконки 192/512, `display: standalone`, цвета).
- Иконки приложения (уже есть `public/icons/mixer-truck.png` — добавить размеры).
- Подключить манифест в `<head>` (через Next.js metadata).
- Для iPhone: обязательна установка «На экран Домой» (iOS 16.4+), иначе Safari
  пуши не отдаёт.

### 2. Service Worker
- `public/sw.js` — обработка события `push` → `self.registration.showNotification`.
- Обработка `notificationclick` → открыть/сфокусировать нужную страницу.
- Регистрация SW на клиенте (`navigator.serviceWorker.register`).

### 3. Подписка Web Push (VAPID)
- Сгенерировать VAPID-ключи (public/private), хранить в env
  (`VAPID_PUBLIC_KEY`, `VAPID_PRIVATE_KEY`).
- Клиент: `registration.pushManager.subscribe({ applicationServerKey })`.
- Таблица в Supabase:
  ```sql
  CREATE TABLE push_subscriptions (
    id BIGSERIAL PRIMARY KEY,
    endpoint TEXT UNIQUE NOT NULL,
    p256dh TEXT NOT NULL,
    auth TEXT NOT NULL,
    role TEXT,           -- 'driver' | 'admin' | 'manager' | 'dispatcher'
    mixer_number TEXT,   -- для водителя: к какому миксеру привязан
    user_id TEXT,
    created_at TIMESTAMPTZ DEFAULT now()
  );
  ```
- API: `POST /api/push/subscribe`, `POST /api/push/unsubscribe`.

### 4. Отправка пушей с сервера
- Библиотека `web-push` (npm).
- Триггер отправки на событиях (переиспользовать существующую логику broadcast):
  - Водителю → новый рейс на его миксер (`order_mixers` INSERT/UPDATE по номеру).
  - Админке → новая заявка + смена статуса (`orders`).
- Функция рассылки: выбрать подписки по роли/миксеру → `webpush.sendNotification`
  по каждому endpoint → чистить «протухшие» подписки (410/404).

## Кнопка в интерфейсе
- В мобильной админке и у водителя: «🔔 Включить пуш-уведомления».
- Запрос разрешения → подписка → сохранение на сервере.
- Индикатор состояния (разрешено / запрещено / не поддерживается).

## Ограничения по платформам (важно)

| Платформа | Работает | Условие |
|---|---|---|
| Android (Chrome) | Да | Разрешить уведомления |
| iPhone (Safari) | Да, но | Только PWA, добавленная «На экран Домой», iOS 16.4+ |
| iPhone (обычная вкладка) | Нет | Ограничение Apple |
