# План: Переезд с Vercel на локальный сервер

## Статус: Отложено (обсуждено, не первоочередное)

## Цель
Перенести приложение с Vercel на собственный локальный сервер / VPS.

## Что нужно сделать при переезде

### 1. node-cron вместо Vercel Cron Jobs
- Установить пакет: `npm install node-cron`
- Создать кастомный `server.js` в корне проекта
- Добавить задачу: каждый день в 00:01 МСК (21:01 UTC) вызывать
  `GET /api/cron/dismiss-notifications` с заголовком `Authorization: Bearer CRON_SECRET`
- Убрать секцию `"crons"` из `vercel.json` (или весь файл, если не нужен)

Пример кода для `server.js`:
```javascript
import cron from 'node-cron';
import { createServer } from 'http';
import next from 'next';

const app = next({ dev: false });
const handle = app.getRequestHandler();

app.prepare().then(() => {
  // 00:01 МСК = 21:01 UTC
  cron.schedule('1 21 * * *', async () => {
    await fetch(`http://localhost:${process.env.PORT || 3000}/api/cron/dismiss-notifications`, {
      headers: { Authorization: `Bearer ${process.env.CRON_SECRET}` }
    });
  });

  createServer(handle).listen(process.env.PORT || 3000, () => {
    console.log('Server ready');
  });
});
```

### 2. Переменные окружения
- Перенести все `.env` переменные на новый сервер:
  - `SUPABASE_URL`
  - `SUPABASE_SERVICE_ROLE_KEY`
  - `SUPABASE_ANON_KEY`
  - `CRON_SECRET`
  - `DADATA_API_KEY`
  - и все остальные из текущего Vercel Dashboard

### 3. Альтернатива node-cron — системный crontab
Если `server.js` не нужен, можно просто добавить одну строку в `crontab -e`:
```
1 21 * * * curl -s -H "Authorization: Bearer ВАШ_CRON_SECRET" http://localhost:3000/api/cron/dismiss-notifications
```

### 4. Процесс-менеджер
- Установить `pm2` для автозапуска при перезагрузке сервера:
  ```bash
  npm install -g pm2
  pm2 start server.js --name concrete-app
  pm2 save && pm2 startup
  ```

### 5. Nginx / обратный прокси
- Настроить Nginx как reverse proxy перед Next.js (порт 3000 → 80/443)
- Настроить SSL через Let's Encrypt (certbot)

## Затронутые файлы
- `server.js` — новый файл (создать при переезде)
- `vercel.json` — убрать секцию `crons`
- `package.json` — добавить `node-cron`, поменять `"start"` скрипт на `node server.js`

## Оценка времени
- node-cron + server.js: ~5 минут
- Перенос env-переменных: ~10 минут
- pm2 + nginx + SSL: ~30–60 минут (зависит от опыта с сервером)
